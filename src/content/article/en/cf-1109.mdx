### Problem B [Nikita and Books](https://codeforces.com/contest/2244/problem/B)

I struggled with this for a while. Instead of thinking about a specific method, judging based on whether it is theoretically possible worked out well.

1. Create a prefix sum array $P_A$ of $A$.
2. Create a prefix sum array $P_B$ of array $B = [1, 2, \ldots, n]$.
3. Output `Yes/No` based on whether $P_A[i] \ge P_B[i] \; (\forall i, 1 \le i \le n)$ is satisfied.

I'll write something like a simple proof:

The simplest form of the array $P$ we want to create is <Marker type="warning">$P[i] = i \; (\forall i)$</Marker>. Therefore, I think it's sufficient to just judge whether that can be created. (It's probably correct.)

### Problem C [Stepan and Permutation](https://codeforces.com/contest/2244/problem/C)

Personally, I found this problem to be the most difficult. If I were to name a similar problem, would it be [Codeforces 1103 - B](https://codeforces.com/contest/2236/problem/B)?
Actually, checking the problem again, it might be completely different.

Well, please forgive me since the line of thinking is similar.

First, let's consider the case where $GCD(x, y) \ne 1$.

1. Let $g = GCD(x, y)$.
2. Output `Yes/No` based on whether $P[i] \equiv i \mod{g} \; (\forall i)$ is satisfied.

Next is the case where $GCD(x, y) = 1$.

Since $x$ and $y$ are coprime, we can change the $mod$.
However, we need to consider whether it can be changed freely. While unproven, I feel like it can be changed if $x + y \le n$.
And that is satisfied by the constraints.

You can solve it using these ideas.

### Problem D [Yaroslav and Productivity](https://codeforces.com/contest/2244/problem/D)

I feel like this has appeared in a contest somewhere every week for the past three weeks. Yes, it's <Marker>Reading Queries in Reverse</Marker>.

1. Create a prefix sum array $P_A$ of $A$.
2. Sort $B$ in ascending order.
3. Add $0$ to the beginning of $B$, and repeat the following operations by taking $(B_{i-1}, B_i)$ from the end of $B$:

    - Add $|P_A[B_i] - P_A[B_{i-1}]|$.

4. Finally, add $P_A[n] - P_A[B_m]$.

In terms of image, it feels similar to [ABC466 - E](https://atcoder.jp/contests/abc466/tasks/abc466_e), but DP is impossible here due to constraints.

If you flip the signs of all values in a certain range, the sign of the total sum in that range flips. Therefore, we manage the total sum of the range using prefix sums.

By looking from the back, you can finalize the values of the ranges seen so far. Therefore, you should greedily take the larger one for a certain range.
